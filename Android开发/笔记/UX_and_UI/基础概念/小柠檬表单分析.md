
# 表单的重要性
使用应用程序或网站的人都有特定的目标。表单通常是用户与目标之间的唯一障碍。在网络和应用程序中，表单仍然是用户交互的基本类型之一。表单通常被视为实现目标的最后一步。用户应该能够快速轻松地填写表单。让我们详细检查一下小柠檬当前的结账表单，它展示了一些常见的表单设计错误。
## 布局
首先，用户需要输入数据的文本字段位于中心，而不是按照最佳实践对齐到左侧。结账按钮不明显，也没有购物篮或关于用户购买内容的信息。

## 只包括必要的字段
越来越多的购物者正在使用智能手机研究潜在购买商品。与此同时，更重要的问题是这些用户是否愿意在移动设备上购买。每个人都回答过那些令人头疼的问题，比如“你是怎么听说我们的？”虽然这些问题可能对卖家有用，但对买家来说却毫无意义。

虽然这些问题在桌面上很烦人，但在移动设备上可能会致命。卢克·罗布列夫斯基在他的优秀著作《Web Forms: Filling in The Blanks》中指出：

“你在 Web 表单中问用户的任何问题都需要他们解析问题、构思回答，然后在表单上提供的适当位置输入答案。对于你所问的每个问题都要保持警惕，这样你就可以删除那些绝对不必要的问题，或者在更好的时间或地点提问，或者可以自动推断。而你问的问题越少，人们完成表单的速度和便捷性就越高。”

小柠檬的结账流程展示了一个简单体验如何迅速变得复杂。页面显示了不必要的字段，比如中间名、晚间电话和移动电话，同时将字段分开，地址字段有三行，城市字段（而不仅仅是邮政编码）还要求用户重新输入他们的电子邮件地址。

## 字符限制
字符限制是一种技术限制，在开发者和设计师合作期间应该被提出。系统需要设置下限和上限字符限制的原因有两个：安全性和存储。

应对输入进行验证，以确保其类型、长度、格式和范围正确。因此，当用户登录以访问信息时，可以将其与数据库进行验证。允许用户自由输入字段的漏洞会导致数据不准确，并可能导致机器人入侵您的系统。

存储中使用最大字符限制是因为不可能存储一个合法长度为 1,073,741,823 字节的名称。世界上最长的名字只有 747 个字符。
### 字符限制：解决方案
去除最小字符数的要求。实施一个适用于数据库中绝大多数名称的最大字符要求-50 个字符是一个很好的起点。为什么要设限呢？字符限制是一种技术限制，应该在开发者和设计师合作期间被提出。系统需要设置下限和上限字符限制以确保安全性和存储。

## 最后的想法
使用互联网最令人沮丧的一点就是表单验证。如果用户不清楚自己做错了什么，而且没有简单的方法来修复，他们很可能会放弃并转向其他网站填写表单。
